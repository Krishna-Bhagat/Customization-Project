# Database Schema Documentation

## Engine

- PostgreSQL (Supabase-compatible)
- Connection managed via `backend/src/db/pool.js`
- Startup schema migration/bootstrap in `backend/src/db/init.js`

## Tables

### `categories`

- `id` (PK)
- `name` (unique)
- `slug` (unique)
- `supports_sizes` (boolean)
- `allowed_sizes` (text[])
- `customization_sides` (text[])
- `created_at`
- `updated_at`

### `products`

- `id` (PK)
- `name`
- `description`
- `category`
- `price`
- `image_url` (primary display fallback)
- `image_urls` (legacy-compatible list)
- `gallery_images` (customer-facing display images)
- `available_sizes` (text[])
- `created_at`

### `product_sides`

- `id` (PK)
- `product_id` (FK -> products.id, cascade delete)
- `side_name`
- `side_key`
- `mockup_image` (customizer-only side mockup)
- `printable_area_x`
- `printable_area_y`
- `printable_area_width`
- `printable_area_height`
- `created_at`
- unique (`product_id`, `side_key`)

## Data Model Notes

- Customer gallery images are separated from side-specific customization mockups.
- Side rows support dynamic product side combinations per category.
- Printable coordinates are persisted as integer columns and generated by visual editor in admin UI.

## Bootstrapping Behavior

On backend startup:

- creates missing tables/columns
- backfills category defaults
- backfills product image arrays from legacy `image_url`
- normalizes missing category side/size config
