# Database Schema Documentation

## Engine

- PostgreSQL (Supabase-compatible)
- Connection managed via `backend/src/db/pool.js`
- Startup schema migration/bootstrap in `backend/src/db/init.js`

## Tables

### `categories`

- `id` (PK)
- `name` (unique)
- `slug` (unique)
- `supports_sizes` (boolean)
- `allowed_sizes` (text[])
- `customization_sides` (text[])
- `created_at`
- `updated_at`

### `products`

- `id` (PK)
- `name`
- `description`
- `category`
- `price`
- `gallery_images` (customer-facing display images)
- `search_slugs` (keyword slug array for storefront search)
- `available_sizes` (text[])
- `created_at`

### `product_sides`

- `id` (PK)
- `product_id` (FK -> products.id, cascade delete)
- `side_name`
- `side_key`
- `mockup_image` (customizer-only side mockup)
- `printable_area_x`
- `printable_area_y`
- `printable_area_width`
- `printable_area_height`
- `created_at`
- unique (`product_id`, `side_key`)

### `users`

- `id` (PK)
- `full_name`
- `phone` (unique, 10 digits)
- `email` (nullable, unique when present)
- `password_hash`
- `created_at`
- `updated_at`

### `addresses`

- `id` (PK)
- `user_id` (FK -> users.id, cascade delete)
- `province`
- `district`
- `municipality_city`
- `ward_number`
- `street_tole`
- `is_default`
- `created_at`
- `updated_at`

### `cart_items`

- `id` (PK)
- `user_id` (FK -> users.id)
- `product_id` (FK -> products.id)
- `selected_size`
- `quantity`
- `selected_sides` (text[])
- `customization_state` (jsonb)
- `preview_image`
- `product_snapshot` (jsonb)
- `unit_price`
- `created_at`
- `updated_at`

### `drafts`

- `id` (PK)
- `user_id` (FK -> users.id)
- `product_id` (FK -> products.id)
- `selected_size`
- `quantity`
- `selected_sides` (text[])
- `active_side`
- `canvas_states` (jsonb)
- `design_exports` (jsonb)
- `preview_image`
- `saved_at`
- `created_at`
- `updated_at`

Unique key: (`user_id`, `product_id`)

### `orders`

- `id` (PK)
- `user_id` (FK -> users.id)
- `order_number` (unique)
- `name_snapshot`
- `phone_snapshot`
- `email_snapshot`
- `address_snapshot` (jsonb)
- `status` (`pending|confirmed|dispatched|delivered`)
- `subtotal_amount`
- `shipping_amount`
- `total_amount`
- `created_at`
- `updated_at`

### `order_items`

- `id` (PK)
- `order_id` (FK -> orders.id, cascade delete)
- `product_id` (FK -> products.id, nullable on delete)
- `product_name`
- `category`
- `size`
- `quantity`
- `unit_price`
- `line_total`
- `selected_sides` (text[])
- `design_urls` (jsonb)
- `customization_data` (jsonb)
- `created_at`

## Data Model Notes

- Customer gallery images are separated from side-specific customization mockups.
- Side rows support dynamic product side combinations per category.
- Printable coordinates are persisted as integer columns and generated by visual editor in admin UI.
- `gallery_images` is the product display source of truth in DB.
- API payloads still include `imageUrl` and `imageUrls` keys derived from `gallery_images` for frontend compatibility.

## Bootstrapping Behavior

On backend startup:

- creates missing tables/columns
- backfills category defaults
- migrates any legacy `image_url` / `image_urls` data into `gallery_images`
- removes legacy `image_url` and `image_urls` columns after migration
- normalizes missing category side/size config
